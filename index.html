1.¿Qué es un Dato?

Un dato es una representación simbólica de un valor, atributo o una característica cuantitativa o cualitativa de una entidad o evento. Los datos son la materia prima para la generación de información y se presentan en formas como números, letras, palabras, o símbolos.

2.¿Qué es información?

La información es el resultado de procesar, organizar y estructurar datos de tal manera que tengan significado y utilidad para quien la recibe. La información proporciona contexto y significado a los datos, permitiendo la toma de decisiones y la resolución de problemas.

3.¿Diferencia entre dato e información?

Dato: Es un elemento sin procesar, sin un contexto que lo haga significativo. Ejemplo: "30", "rojo", "Juan

Información: Es el resultado de procesar datos de tal manera que adquieran significado en un contexto específico. Ejemplo: "Juan tiene 30 años" o "El coche rojo pertenece a Juan".

4.¿Tipos de datos?

Numéricos: Enteros (int), Flotantes (float)

Texto: Cadenas de caracteres (string)

Booleanos: Verdadero (true) o Falso (false)

Fecha y hora: Representación de momentos en el tiempo

Binarios: Datos en formato binario (imágenes, archivos)

5.¿Qué es un modelado de Datos?

El modelado de datos es el proceso de crear un modelo que represente los datos y sus relaciones dentro de un sistema de información. Este modelo actúa como un plano para el diseño y la implementación de una base de datos, asegurando que los datos se estructuren y se gestionen de manera coherente y eficiente.

7.¿Qué es el modelo relacional?

El modelo relacional es un enfoque para la gestión de bases de datos en el que los datos se organizan en tablas (o relaciones). Cada tabla está compuesta por filas (tuplas) y columnas (atributos). Las tablas pueden estar relacionadas entre sí mediante claves primarias y claves foráneas, lo que permite establecer relaciones entre los datos de diferentes tablas.

9.¿Qué es una entidad?

Una entidad es cualquier objeto o concepto del mundo real que tiene una existencia independiente y puede ser identificado de manera única. Ejemplos de entidades pueden ser una persona, un coche, un libro, etc.

10.¿Qué es un atributo y sus tipos?

Un atributo es una propiedad o característica de una entidad. Los tipos de atributos incluyen:

Simples: No se pueden dividir en partes más pequeñas. Ejemplo: Edad

Compuestos: Se pueden dividir en partes más pequeñas. Ejemplo: Dirección (calle, ciudad, código postal).

Derivados: Se pueden calcular a partir de otros atributos. Ejemplo: Edad a partir de la fecha de nacimiento.

Multivalorados: Pueden tener múltiples valores. Ejemplo: Teléfonos (un cliente puede tener varios números de teléfono).

11.¿Qué es una relación?

Una relación es una asociación entre dos o más entidades. Define cómo las entidades están conectadas entre sí. Por ejemplo, una relación entre las entidades "Estudiante" y "Curso" podría ser "Inscrito en", indicando qué estudiantes están inscritos en qué cursos.

12.¿Qué son las llaves primarias, foránea y sus características?

Clave primaria (Primary Key): Es un atributo o conjunto de atributos que identifica de manera única cada fila en una tabla. Debe ser única y no nula. Ejemplo: Número de identificación del empleado.

Clave foránea (Foreign Key): Es un atributo en una tabla que se refiere a la clave primaria de otra tabla, estableciendo una relación entre las dos tablas. Permite mantener la integridad referencial. Ejemplo: ID del departamento en la tabla de empleados que hace referencia a la tabla de departamentos.

14.¿Qué es la cardinalidad o mapeo?

La cardinalidad se refiere al número de instancias de una entidad que pueden o deben estar asociadas con una instancia de otra entidad en una relación. Los tipos de cardinalidad incluyen:

Uno a uno (1:1): Cada instancia de una entidad se asocia con una única instancia de otra entidad.

<|p>: Una instancia de una entidad se puede asociar con múltiples instancias de otra entidad.

13.¿Qué son las reglas del Negocio?

Las reglas del negocio son directrices o restricciones que definen o limitan ciertos aspectos del comportamiento de un negocio. Estas reglas aseguran que las operaciones de la base de datos se alineen con las políticas y procedimientos del negocio.
